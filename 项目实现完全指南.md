## 1. 项目总览与架构

### 1.1 项目定位

AeroVLA 是一套基于 Vision-Language-Action (VLA) 范式的无人机自主飞行控制系统，核心思路是：

```
视觉输入(双摄像头) + 语言指令(自然语言) → VLA模型推理 → 3-DoF连续控制信号 → AirSim飞控执行
```

项目基于 ECCV 2026 论文 AeroVLA (原名 AerialVLA)，采用 OpenVLA-7B 作为基础模型，通过 LoRA 微调适配无人机导航场景。

### 1.2 核心能力

| 能力 | 实现方式 |
|------|---------|
| 双视角视觉感知 | 前视 + 下视摄像头，拼接为 224×448 mosaic |
| 自然语言指令理解 | 开放式语言输入（如 "a red car parked on the street"） |
| 3-DoF 连续控制 | forward (0-5m), down (-5~5m), yaw (-1.1~1.1 rad) |
| 自主着陆判断 | 模型内禀 LAND 信号，无需外部检测器 |
| 模糊方向提示 | 基于机载传感器计算 "straight ahead" / "to your left" 等 |
| 多模型后端 | OpenVLA-7B+LoRA (本地/远程) / Ollama VLM (llava:7b等) |
| 商用地勤系统 | FastAPI + WebSocket + Leaflet 地图 SPA |

### 2.2 数据流

#### 2.2.1 VLA 闭环数据流

```
1. DualCameraCapture 每 200ms 采集一帧
   → 保存 camera_front.jpg, camera_down.jpg 到 vla_shared/

2. AeroVLAProGUI._loop() 每步:
   a. 读取 camera_front.jpg + camera_down.jpg (BGR numpy)
   b. 读取 status.json 获取无人机位置/姿态
   c. 计算 direction_hint (基于当前位置与目标的角度)
   d. 调用 VLAEngineHub.vla_infer(front, down, instruction, direction_hint)
      → 引擎选择: OpenVLA / Ollama / Remote
      → 返回 {fwd, down, yaw, land, infer_time}
   e. 将 3-DoF 信号转换为 ENU 速度:
      vx_body = min(fwd / dur, spd_limit)
      vx_enu = vx_body * cos(yaw)
      vy_enu = vx_body * sin(yaw)
      vz = down / dur
   f. 写入 command.json:
      {"stage": "vla_action", "command": {"action": "vla_action",
       "vx": ..., "vy": ..., "vz": ..., "delta_yaw": ..., "duration": ...}}

3. AirSimOffboardController 每 100ms 检查 command.json
   → 解析 action 类型
   → vla_action: 调用 moveByVelocityAsync(vx, vy, -vz, duration, YawMode)
   → takeoff/land/hover/move_to: 调用对应 AirSim API
   → 写回 status.json (位置/姿态/阶段)
```

#### 2.2.2 前后端数据流

```
AirSim 控制器 → POST /api/telemetry/ingest → FastAPI → SQLite (FlightLog)
                                        → WebSocket 推送 → Web 前端实时更新

Web 前端 → POST /api/missions → FastAPI → SQLite (Mission)
         → POST /api/drones   → FastAPI → SQLite (Drone)
         → WebSocket command  → vla_shared/command.json → AirSim 控制器
```

### 3.2 动作空间设计

AeroVLA 将连续控制信号离散化为 99 个 bin（与 OpenVLA 训练一致）：

```python
# aerovla_inference.py Line 44-50
NUM_BINS = 99
NORM_STATS = {
    'forward': {'min': 0.0, 'max': 5.0},    # 前进距离 (m)
    'down':    {'min': -5.0, 'max': 5.0},   # 垂直移动 (m, 正=上升)
    'yaw':     {'min': -1.1, 'max': 1.1}    # 偏航角变化 (rad)
}
```

**离散化公式** (训练时):
```python
bin_val = round((continuous_val - min) / (max - min) * (NUM_BINS - 1))
```

**反离散化公式** (推理时):
```python
# aerovla_inference.py Line 302-307
def _dequantize(self, bin_val: int, axis: str) -> float:
    stats = self.NORM_STATS[axis]
    vmin, vmax = stats['min'], stats['max']
    bin_clamped = max(0, min(self.NUM_BINS - 1, bin_val))
    return (bin_clamped / (self.NUM_BINS - 1)) * (vmax - vmin) + vmin
```

### 3.3 Prompt 格式

训练和推理时使用统一的 prompt 格式：

```
<image>
Fly {direction_hint}and find the target. {instruction}
Action: 
```

其中:
- `<image>` 是图像占位符
- `direction_hint` 是模糊方向提示 (如 "straight ahead ", "to your left ")
- `instruction` 是目标描述 (如 "a red car parked on the street")

模型输出示例:
```

```
或着陆时:

### 3.5 方向提示计算

```python
# aerovla_gui_pro.py Line 41-61
def compute_direction_hint(pos, quat, target):
    """计算从当前位置到目标的方向提示（ENU坐标系）。"""
    dx, dy, dz = target[0]-pos[0], target[1]-pos[1], (target[2] or 5.0)-pos[2]
    r = R.from_quat([quat[1], quat[2], quat[3], quat[0]])
    fwd = r.apply([1, 0, 0])
    dot_fwd = dx * fwd[0] + dy * fwd[1]
    cross = fwd[0] * dy - fwd[1] * dx
    dist = np.sqrt(dx**2 + dy**2)
    if dist < 1: return "target directly below or very close"
    parts = [f"distance {dist:.1f}m"]
    if dz > 2: parts.append("target is above you")
    elif dz < -2: parts.append("target is below you")
    if dot_fwd > 0.7: parts.append("straight ahead")
    elif dot_fwd < -0.7: parts.append("behind you")
    elif cross > 0: parts.append("to your left")
    else: parts.append("to your right")
    return ", ".join(parts)
```

## 4. VLA 训练详解

### 4.1 训练方法: LoRA 微调

AeroVLA 采用 LoRA (Low-Rank Adaptation) 微调 OpenVLA-7B 基础模型：

```
OpenVLA-7B (冻结) + LoRA Adapter (可训练)
  ↓
  LoRA 只训练低秩矩阵 W = W₀ + BA
  其中 B ∈ R^(d×r), A ∈ R^(r×k), r << min(d, k)
  ↓
  训练参数量: ~100MB (vs 全量 15GB)
  训练显存: ~16GB (vs 全量 ~40GB)
```

### 4.2 训练数据

训练数据来自 TravelUAV benchmark:

| 数据集 | 说明 |
|--------|------|
| `dataset_raw/` | 原始数据 (多城市场景) |
| `data/uav_dataset/seen_valset_splits/` | 已见场景验证集 (11 个地图) |
| `data/uav_dataset/unseen_map_valset_splits/` | 未见地图验证集 |
| `data/uav_dataset/unseen_object_valset_splits/` | 未见目标验证集 |
| `data/meta/map_spawnarea_info.json` | 地图生成区域信息 |
| `data/meta/object_description.json` | 目标物体描述 |

训练数据格式 (每条轨迹):
```json
{
  "instruction": "Fly 30 degrees from you. Please control the drone to find a red car. ...",
  "trajectory": [...],  // 专家轨迹
  "object": {"asset_name": "AA_RedCar", ...},
  "object_position": [x, y, z]
}
```

### 4.4 DAgger 训练流程

AeroVLA 使用 DAgger (Dataset Aggregation) 策略进行训练:

```
Round 0: 收集专家轨迹 (oracle policy)
  → 训练 LoRA 第 0 轮

Round 1: 用第 0 轮模型在仿真中运行
  → 收集模型轨迹 + 专家修正
  → 合并到训练集 (dagger_update_size=8000)
  → 继续训练 LoRA 第 1 轮

... 重复 dagger_it 轮
```

### 5.2 本地推理引擎 (aerovla_inference.py)

```python
    front_img=front_bgr,    # BGR numpy (H×W×3)

    direction_hint="straight ahead"

```

推理流程:
```
1. BGR → RGB → resize(224×224) → mosaic(224×448)
2. image_processor → pixel_values tensor
3. 构造 prompt: "<image>\nFly {dir}and find the target. {instr}\nAction: "
4. tokenizer → input_ids
5. model.generate(max_new_tokens=20, do_sample=False)
6. 解析输出: "Action: 23 50 49" → dequantize → (fwd, down, yaw)
7. 检查 LAND 信号
```

### 6.4 4-bit 量化加载 (含 monkey-patch)

```python
# aerovla_inference.py — 4-bit 加载的关键代码

# 4-bit 量化下 peft 0.11.1 和 transformers 有兼容性 bug:
# 多处调用 requires_grad_(True)，但 Params4bit 不是浮点类型会报 RuntimeError
# 修复: patch torch.nn.Module.requires_grad_ 跳过非浮点参数

import torch.nn as nn
from torch.nn import Parameter

def _safe_requires_grad_(self, requires_grad=True):
    for p in self.parameters():
        if p.dtype in (torch.float16, torch.bfloat16, torch.float32,
                       torch.float64, torch.complex64, torch.complex128):
            p.requires_grad_(requires_grad)
    return self

nn.Module.requires_grad_ = _safe_requires_grad_

# 加载 4-bit 模型
from transformers import BitsAndBytesConfig
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    llm_int8_skip_modules=["projector"],  # 跳过 projector 层
)
model = AutoModelForVision2Seq.from_pretrained(
    base_model_path,
    torch_dtype=torch.bfloat16,
    quantization_config=bnb_config,
    low_cpu_mem_usage=True,
    trust_remote_code=True,
)

# 加载 LoRA (量化模式下不 resize_token_embeddings)
from peft import PeftModel
model = PeftModel.from_pretrained(model, lora_path, is_trainable=False)

# 修复 4-bit 量化后 inv_freq buffer 留在 CPU 的问题
for name, module in model.named_modules():
    if hasattr(module, 'inv_freq'):
        module.inv_freq = module.inv_freq.to(self.device)
```

### 6.8 AWQ 量化在 Windows 不可用的原因

```

  quantizers/quantizer_awq.py → validate_environment() → is_gptqmodel_available()
  integrations/awq.py → from gptqmodel.utils.model import hf_gptqmodel_post_init

gptqmodel 安装失败链:
  gptqmodel → pypcre → git clone PCRE2 → curl 28 (Connection reset)
  无 Windows 预编译 wheel
  autoawq → triton (仅 Linux)

结论: AWQ 量化在 Windows + transformers 5.9.0 下完全不可用
```

### 8.3 VLA 速度控制

```python
# airsim_offboard_controller.py Line 358-390
def _do_vla_action(self, cmd):
    """执行 VLA 模型输出的单步速度指令"""
    vx = float(cmd.get('vx', 0))      # ENU X 速度 (m/s)
    vy = float(cmd.get('vy', 0))      # ENU Y 速度 (m/s)
    vz = float(cmd.get('vz', 0))      # ENU Z 速度 (m/s, 正=上升)
    dyaw = float(cmd.get('delta_yaw', 0))  # 偏航角速率
    duration = float(cmd.get('duration', 0.5))

    # 限制速度
    max_speed = 5.0
    vx = max(-max_speed, min(max_speed, vx))
    vy = max(-max_speed, min(max_speed, vy))
    vz = max(-max_speed, min(max_speed, vz))

    # AirSim 使用 NED 坐标系 (z 向下为正)
    # ENU → NED: z 取负
    self.client.moveByVelocityAsync(
        vx, vy, -vz, duration,
        airsim.DrivetrainType.MaxDegreeOfFreedom,  # 机体坐标系
        airsim.YawMode(False, dyaw),               # yaw_rate 模式
        vehicle_name=self.drone_name
    )
```

### 8.4 3-DoF 到 ENU 速度转换

```python
# aerovla_gui_pro.py Line 936-951
# VLA 输出: fwd (body frame), down, yaw
# 转换为 ENU 速度: vx_enu, vy_enu, vz

dur = self.dur  # 每步时长
vx_body = min(fwd / dur, self.spd)   # body frame 前向速度
vz = min(down / dur, self.spd)       # 垂直速度
dyaw = yaw                           # 偏航角速率

# 从四元数获取当前偏航角
r = R.from_quat(quat)
euler = r.as_euler("zyx")
cy = euler[0]  # yaw angle

# body → ENU 转换
vx_enu = vx_body * np.cos(cy)
vy_enu = vx_body * np.sin(cy)
```

### 9.2 数据库模型 (aerovla_models.py)

```
Users (用户)
  ├── id, username, hashed_password, role (admin/operator/viewer)
  ├── display_name, email, is_active
  ├── created_at, last_login
  └── 1:N → Missions

Drones (无人机)
  ├── id, drone_id, name, model, firmware
  ├── status (online/offline/flying/error)
  ├── battery, position_lat/lng, altitude, heading, speed
  ├── connected_since, last_telemetry
  └── 1:N → Missions, FlightLogs

Missions (任务)
  ├── id, name, description, mission_type (waypoint/search/patrol)
  ├── status (draft/planned/executing/completed/aborted)
  ├── waypoints (JSON: [{lat, lng, alt, action, speed, heading}])
  ├── vla_instruction (自然语言 VLA 指令)
  ├── drone_id, created_by
  ├── created_at, started_at, completed_at
  ├── flight_time_s, distance_m
  └── 1:N → FlightLogs

FlightLogs (飞行日志)
  ├── id, mission_id, drone_id
  ├── timestamp, event_type (telemetry/vla_action/error/command)
  ├── position_lat/lng, altitude, heading, speed, battery
  ├── vla_fwd, vla_down, vla_yaw, vla_confidence
  ├── message
  └── data (JSON, 扩展字段)
```

### 9.4 WebSocket 遥测推流

```python
# aerovla_server.py
_ws_clients: set[WebSocket] = set()
_latest_telemetry: dict = {}  # drone_id → telemetry

async def _broadcast_loop():
    """每 1s 广播最新遥测给所有 WebSocket 客户端。"""
    while True:
        if _ws_clients and _latest_telemetry:
            payload = json.dumps({"type": "telemetry", "data": _latest_telemetry})
            dead = set()
            for ws in _ws_clients:
                try: await ws.send_text(payload)
                except: dead.add(ws)
            _ws_clients.difference_update(dead)
        await asyncio.sleep(1)

@app.websocket("/ws")
async def websocket_endpoint(ws: WebSocket):
    await ws.accept()
    _ws_clients.add(ws)
    # ... 接收/处理消息 ...
```

## 11. 故障排查

### 11.1 模型加载失败

| 错误 | 原因 | 解决 |
|------|------|------|
| `config.json not found` | 模型路径错误 | 检查 `BASE_MODEL_PATH` 指向包含 config.json 的目录 |
| `adapter_config.json not found` | LoRA 路径错误 | LoRA 必须在 `weight-lora/aerial_vla/` 子目录 |
| `CUDA out of memory` | 显存不足 | 换 4-bit 量化，或用远程服务器 |
| `Params4bit requires_grad_ RuntimeError` | 4-bit + peft bug | 代码已内置 monkey-patch，确认 aerovla_inference.py 最新 |
| 模型输出始终为 LAND | 4-bit 量化退化 | 改用 8-bit (`load_in_8bit=True`) |
| 加载时卡住不动 | 可能在下载文件 | 设置 `HF_HUB_OFFLINE=1` |
| `Loading AWQ requires gptqmodel` | AWQ 在 Windows 不可用 | 改用 float16 或 8-bit 量化 |

### 11.2 AirSim 相关

| 错误 | 原因 | 解决 |
|------|------|------|
| `Connection refused` | AirSim 未启动 | 先启动 AirSimNH，再启动控制器 |
| 摄像头黑屏 | 相机名称不匹配 | 检查 settings.json 中的相机名 |
| 无人机不动 | API 控制未启用 | `client.enableApiControl(True)` |
| 飞行抖动/振荡 | 速度控制频率过高 | 增大 `MANEUVER_UPDATE_INTERVAL` |
| 中文路径报错 | cv2.imwrite 不支持中文 | 使用 `cv2.imencode + tofile` |

### 11.3 Ollama 相关

| 错误 | 原因 | 解决 |
|------|------|------|
| `无法连接 Ollama` | 服务未启动 | 运行 `ollama serve` |
| `模型未找到` | 模型未拉取 | `ollama pull llava:7b` |
| JSON 解析失败 | 模型输出格式不规范 | 适配器有关键词回退解析 |
| 推理太慢 | 模型太大 | 换轻量模型 `moondream` |

### 11.4 前后端相关

| 错误 | 原因 | 解决 |
|------|------|------|
| 登录 401 | 密码错误 | 默认 admin/admin888 |
| WebSocket 不推数据 | 没有遥测上报 | 调用 `POST /api/telemetry/ingest` |
| 地图不显示 | Leaflet CDN 加载失败 | 检查网络连接 |
| CORS 错误 | 跨域未配置 | 已配置 `allow_origins=["*"]` |
